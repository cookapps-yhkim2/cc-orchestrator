# unity-gamedesign

게임 디자인 문서화 스킬입니다.
밸런싱 테이블, 시스템 설계, AI 행동트리 템플릿을 제공합니다.

## 트리거

- "밸런싱 테이블 만들어줘"
- "AI 행동트리 설계해줘"
- "게임 시스템 설계"
- "스탯 테이블 만들어줘"
- "레벨 디자인 문서"
- "게임 디자인 문서 작성"

## 참조 문서

1. **`references/balancing-templates.md`** — 스탯 테이블, 경험치 곡선, 드롭 테이블, 경제 밸런싱 템플릿
2. **`references/behavior-tree-guide.md`** — 행동트리 기본 개념, 노드 타입, Unity 구현 패턴

## 역할

이 스킬은 코드를 직접 작성하지 않습니다. 대신:

1. **게임 시스템을 구조화**합니다 — 막연한 아이디어를 표/수식/트리로 정리
2. **밸런싱 데이터를 설계**합니다 — ScriptableObject에 넣을 수치 체계를 잡아줌
3. **AI 행동을 설계**합니다 — 행동트리나 상태머신의 로직 구조를 정의
4. **Implementer에게 전달할 스펙**을 생성합니다

## 수행 작업

### 밸런싱 테이블 요청 시

1. 사용자에게 게임의 핵심 시스템을 확인합니다:
   - 어떤 종류의 밸런싱? (캐릭터 스탯, 무기, 적, 경제, 레벨업 등)
   - 게임의 목표 플레이 시간 / 난이도 곡선은?
   - 참고하는 레퍼런스 게임이 있는가?

2. `references/balancing-templates.md`를 참고하여 적합한 템플릿을 선택합니다.

3. 초안을 작성하여 사용자에게 보여주고, 수치를 조정합니다.

4. 최종 결과를 `.unity-orc/plans/` 또는 별도 문서에 저장합니다.

### AI 행동 설계 요청 시

1. AI 캐릭터의 역할과 행동 목록을 확인합니다:
   - 어떤 종류의 AI? (적, NPC, 보스, 동료 등)
   - 기본 행동 목록은? (순찰, 추적, 공격, 도주 등)
   - 특수 행동이 있는가? (패턴 공격, 페이즈 전환 등)

2. `references/behavior-tree-guide.md`를 참고하여 행동트리를 설계합니다.

3. 트리 다이어그램 + 각 노드 설명을 문서로 작성합니다.

### 시스템 설계 요청 시

게임 시스템(인벤토리, 퀘스트, 대화, 전투 등)에 대해:

1. **시스템 개요**: 무엇을 하는 시스템인가
2. **핵심 데이터 구조**: 어떤 데이터를 관리하는가
3. **주요 흐름**: 플레이어 행동 → 시스템 반응
4. **다른 시스템과의 연결**: 의존성 관계
5. **ScriptableObject 설계**: 어떤 데이터를 SO로 분리하는가

## 문서 형식

게임 디자인 문서는 다음 구조를 따릅니다:

```markdown
# {시스템명} 설계 문서

## 개요
{시스템의 목적과 핵심 기능 2-3줄}

## 핵심 메카닉
{플레이어 관점에서 어떻게 동작하는가}

## 데이터 구조
{클래스, ScriptableObject, enum 등의 설계}

## 수치 테이블
{밸런싱 수치가 있다면}

## 흐름도
{주요 로직 흐름을 텍스트 다이어그램으로}

## Implementer 참고사항
{구현 시 주의할 점, 권장 패턴}
```

## 완료 조건

- [ ] 사용자의 게임 디자인 의도를 정확히 파악함
- [ ] 구조화된 문서가 생성됨 (표, 수식, 다이어그램 포함)
- [ ] Implementer가 코드로 옮길 수 있을 만큼 구체적임
- [ ] 밸런싱 수치에 대한 근거가 명시됨 (공식이나 레퍼런스)
